# Peer Lab Exercise: Abstraction & Polymorphism (Java 2026)

## Objective

The goal of this lab is to **apply Abstraction and Polymorphism using modern Java practices (2026)**.  
Students will collaboratively design **interfaces and abstract classes**, and use each other's work to demonstrate:

- Overriding
- Overloading
- Dynamic binding
- Interface-driven design

This lab enforces:
- **Abstraction-first design**
- **Reusable contracts**
- **Object collaboration**
- **Professional Git workflow (branch → PR → main)**

---

## Challenge Goal

By completing this exercise you should demonstrate:

- Proper use of **abstract classes**
- Proper use of **interfaces (with default methods)**
- Implementation of **method overriding**
- Implementation of **method overloading**
- Understanding of **runtime polymorphism (dynamic binding)**
- Strong **low coupling and high cohesion**
- Professional **Git workflow**

---

## Tasks

### 1. Create a New Branch (Required)

Create a new branch from `main`.

```
abspoly-<your-name>
```

Make sure your branch is updated with the latest `main`.

---

### 2. Create Your Own Abstraction (Required)

Each student must create:

- **1 Abstract Class**
- **1 Interface**

#### Abstract Class Requirements:
- At least **1 abstract method**
- At least **1 concrete method**

#### Interface Requirements:
- At least **1 abstract method**
- At least **1 default method**

---

### 3. Commit, Push, PR, and Sync (Required)

- Commit your work to your branch  
- Push your branch to GitHub  
- Create a **Pull Request to `main`**  
- After PR creation, **pull updates from `main` and merge into your branch**

Commit example:

```
Added abstract class and interface
```

---

### 4. Use Teammate Abstractions

Use:
- **1 abstract class from a teammate**
- **1 interface from a teammate**

---

### 5. Demonstrate Method Overriding

Override:
- Abstract method from abstract class
- Interface method

---

### 6. Demonstrate Method Overloading

Create at least **one overloaded method**

---

### 7. Create a Demo Program

```
AbstractionPolyDemo_Surname.java
```

Must show:
- Overriding
- Overloading
- Dynamic binding
- Use of abstractions

---

### 8. Demonstrate Dynamic Binding

Use abstraction references with multiple implementations.

---

### 9. Add Code-Based Analysis (Don't create a new file, use the files you already created)

Add comments inside your code answering:

- What abstract class did you create?
- What interface did you create?
- What methods did you override?
- What methods did you overload?
- Where does dynamic binding occur in your code?
- Which part shows polymorphism?
- How does your design achieve low coupling?
- How does your design achieve high cohesion?

---

### 10. Commit and Push All Work

Include all required files.

---- 
By the end of the lab activity, each student should have the following files:
1. 1 Abstract Class you created
2. 1 Interface you created
3. At least 2 Concrete Class Implementations of a teammate's Abstract Class
4. At least 2 Concrete Class Implementations of a teammate's Interface
5. 1 Demo file that contains main method and shows method overloading and overriding using files created

---

### 11. Create a Pull Request (Required)

Create a PR to `main`.

---

Think of **at least 2 things you each could have done better**.
