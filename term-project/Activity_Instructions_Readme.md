# 🧪 Lab Exercise: Term Project Planning  
### Defining Real-World Use Cases

---

## 🎯 Objective

The goal of this activity is to help your team **decide on a possible term project** and clearly define **what your application will do**.

This **first planning session focuses only on identifying the use cases of the system**.

⚠️ The plan **does not need to be perfect**. It will be **refined and improved in future sessions**.

Each team has **14 members**, and the project will run across a **14-week term**.

---

# 📝 Activity Tasks

---

## 1️⃣ Form Your Team

Confirm your **team of 14 members**.

Assign the following roles:

| Role | Responsibility |
|-----|-----|
| Facilitator | Leads the discussion |
| Documenter | Writes and organizes the ideas |

---

## 2️⃣ Brainstorm Application Ideas

As a team, discuss **possible applications** you can build for the term project.

### Example Ideas

- Student Task Manager  
- Library System  
- Event Registration System  
- Inventory Manager  
- Clinic Appointment System  
- Campus Lost and Found System  

### Your Application Should

✔ Solve a **real problem**  
✔ Be **realistic to complete within 14 weeks**  
✔ Contain **multiple features** that different members can work on  

📌 **Task:**  
List **at least 3 possible project ideas**.

---

## 3️⃣ Select One Project Idea

Discuss the proposed ideas and **vote on one project** that the team will build for the term.

### Record the following:

- **Project Name**
- **Short Description**
- **Target Users**  
  *(students, teachers, businesses, etc.)*

---

## 4️⃣ Identify the Main Users of the System

List the types of **users who will interact with the system**.

### Examples

- Admin
- User
- Customer
- Student
- Staff

📌 A system may have **more than one user type**.

---

## 5️⃣ Define the Use Cases

For each user type, list the **actions they should be able to perform** in the application.

Focus on **what the user does**, not how it is implemented.

### Example Actions

- Register an account
- Login to the system
- Add a record
- View records
- Update information
- Delete records
- Search for items
- Generate reports

📌 Your team should identify **at least 8–12 use cases**.

---

## 6️⃣ Write Simple Use Case Statements

Write each use case using **simple sentence format**.

### Example Format
The user can create an account.
The admin can view all users.
The user can update their profile.
The admin can delete records.


⚠️ Avoid technical details.  
Focus only on **what the system allows the user to do**.

---

## 7️⃣ Organize the Use Cases

Group the use cases **by user type**.

### Example

#### Admin

- View all users  
- Delete users  
- Manage records  

#### User

- Register account  
- Login  
- View personal data  
- Update profile  

---

## 8️⃣ Identify the Most Important Use Cases

From the full list, identify the **core features** required for the system to function.

These will become the **main features your team must implement**.

📌 Select **around 5–8 core use cases**.

---

## 9️⃣ Assign Initial Feature Owners

Assign **at least one team member** to each major use case.

These members will **initially be responsible for researching and designing the feature**.

### Example

| Feature | Assigned Member |
|-------|-------|
| User Registration | Member 1 |
| Login System | Member 2 |
| Record Management | Member 3 |

⚠️ This assignment **can change later** as the project evolves.

---

## 🔟 Document Your Work

Create a document containing the following sections:

- Team Members
- Project Name
- Project Description
- User Types
- List of Use Cases
- Core Use Cases
- Initial Feature Owners

---

## 📦 Commit to the Team Repository

Upload your document to your **team GitHub repository** so the entire team can **continue improving it during the term**.

---

# 🔔 Reminder

This is **only the first planning session**.

Your **design, features, and responsibilities may change** as your understanding of the system improves throughout the term.

Focus on **clarity, collaboration, and realistic planning**.
